### inventory-app![](https://media.giphy.com/media/dWesBcTLavkZuG35MI/giphy.gif)---## Inventory App — курсовой проект "IIransition" (на Symfony 6.4 LTS)Проект реализует минимальную систему управления инвентарями в соответствии с требованиями курсовой работы.Основной фокус сделан на:- архитектуру приложения- корректную модель данных- контроль доступа (ACL)- расширяемость и поддерживаемостьВизуальная часть intentionally минималистична и не является целью проекта.---## Проверка под пользователямиURL:http://127.0.0.1:8000/login**Демонстрационные аккаунты (Doctrine Fixtures)**Администратор:- Email: `admin@test.com`- Пароль: `admin123`Обычный пользователь:- Email: `user@test.com`- Пароль: `user123`---## Основные возможности- Инвентари (Inventory)- Элементы инвентарей (Inventory Items)- Поддержка кастомных полей- Архитектурная поддержка кастомных ID- Пользователи и роли- Контроль доступа (ACL) через Symfony Security Voters- Обсуждения (Discussion)- Лайки (Likes)- Оптимистическая блокировка (Optimistic Locking)---## Архитектура проектаПроект построен с чётким разделением ответственности:- **Controller** — обработка HTTP-запросов- **DTO** — передача данных между слоями- **Entity** — доменная модель- **Repository** — работа с БД- **Service** — бизнес-логика- **Security / Voter** — контроль доступа (ACL)Бизнес-логика не хранится в шаблонах или контроллерах.---## Контроль доступа (ACL)Контроль доступа реализован **на серверной стороне** с использованием Symfony Security Voters.### Роли и права| Тип пользователя | Права ||------------------|-------|| Guest | Просмотр публичных инвентарей || User | Просмотр чужих публичных инвентарей || Owner | Полный доступ к своему инвентарю || Admin | Полный доступ ко всем инвентарям |### Inventory ACLПрава проверяются через `InventoryVoter`:- INVENTORY_VIEW- INVENTORY_CREATE_ITEM- INVENTORY_EDIT_ITEM- INVENTORY_DELETE_ITEM- INVENTORY_MANAGE_ACCESS- DISCUSSION_WRITEДополнительный доступ реализован через сущность `InventoryAccess`:- READ — только просмотр- WRITE — создание, редактирование, удаление элементовДаже если UI выглядит одинаково, **фактические права пользователей различаются**  и всегда проверяются на backend-уровне.---## Inventory Items (CRUD)Для элементов инвентаря реализовано:- Просмотр- Создание- Редактирование- УдалениеКаждое действие защищено ACL-проверками через `InventoryVoter`.---## Discussion и LikesФункциональность обсуждений и лайков реализована **на backend-уровне**:- polling-архитектура- подсчёт лайков- ACL-контроль на записьUI используется только как демонстрация, без сложных интерактивных элементов.---## UI-подход- UI намеренно минималистичный- Используется нейтральная (серая) цветовая схема- Отсутствуют «цветные светофоры»- Кнопки действий отображаются только при наличии прав- В Twig-шаблонах отсутствует бизнес-логикаИнтерфейс служит исключительно для демонстрации backend-архитектуры.---## Технологии- PHP 8.2- Symfony 6.4 LTS- Doctrine ORM- PostgreSQL- Twig- Symfony Security---## ПримечанияЧасть дополнительных возможностей (сложный UI, полнотекстовый поиск, автосохранение через UI, облако тегов) не реализована.Проект сфокусирован на:- корректной архитектуре- расширяемости- безопасности- в соответствии с требованиями курсовой работы
